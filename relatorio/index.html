Autores: Jorge L. V. Jabczenski & Vinicius T. V. Date

Cliente

O cliente é baseado no cliente normal UDP estudado em aula.

Entretanto ele recebe um argumento extra: o número de mensagens a serem enviadas.

Cada mensagem é composta pelo seu número de sequência, a fim de ser analisada pelo servidor.

Servidor

No lado do servidor temos um timeout definido, pois o servidor não tem como saber quando a última mensagem foi enviada.

Ao receber uma mensagem, ele tem consigo um número esperado de sequência, caso este número seja diferente do número recebido, o servidor então consegue identificar 3 casos:

  • Se o número de sequência recebido for menor do que o esperado, quer dizer que o pacote chegou fora de ordem.
  • Se ocorre o contrário, o número de sequência recebido for maior que o esperado, o pacote também chegou fora de ordem e possivelmente os pacotes anteriores não foram recebidos.
  • Se, por fim, o número for o esperado, o recebimento ocorreu de forma esperada.
